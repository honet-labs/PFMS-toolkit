================================================================================
NAMING CONVENTIONS
================================================================================

Interface naming follows LibreNMS standards:

  GigabitEthernet 0/0/1  →  ge0/0/1
  Ethernet1              →  eth1
  FastEthernet 0/1       →  fe0/1
  VLAN 100               →  vlan100
  Management             →  mgmt
  LoopBack 0             →  lo
  Tunnel 1               →  tun
  Serial 0/0             →  ser0/0
  ATM 0/0                →  atm0/0

This allows easy integration with:
  - LibreNMS
  - Nagios/Icinga
  - Zabbix
  - Prometheus
  - Custom monitoring tools

TROUBLESHOOTING
================================================================================

Issue: Bootstrap doesn't load
  Solution: Check if all required classes exist in app/DiscoveryModules/

Issue: No interfaces discovered
  Solution: Verify SNMP community string and device supports IF-MIB

Issue: Wrong interface names
  Solution: Check ifName (RFC 2096) support on device, may need manual mapping

Issue: Slow discovery
  Solution: Increase SNMP timeout to 5+ seconds, reduce retries

Issue: Vendor-specific metrics missing
  Solution: Verify sysObjectID matches vendor detection prefix
